# Documentation

This documentation is organized by what the reader is trying to do.

- [Tutorials](tutorials/README.md) teach a complete outcome step by step.
- [How-to guides](how-to/README.md) solve focused operational tasks.
- [Reference](reference/README.md) records configuration and command facts.
- [Explanation](explanation/README.md) describes architecture and trust boundaries.
- [Decisions](decisions/README.md) records durable choices and tradeoffs.

Choose a section by reader intent, use kebab-case filenames, and link every new
document from its nearest index. Keep temporary plans in issues.
